# ADR-001: Google News URL 디코딩 및 번역 프레임워크 설계

* **상태 (Status):** Accepted
* **작성자 (Author):** Antigravity (AI Coding Assistant)
* **작성일 (Date):** 2026-06-04

---

## 1. 맥락 (Context)
원래 AgentHub는 Google News RSS 피드를 긁어서 뉴스 카드 목록을 만들었습니다. 그러나 RSS 피드에 들어있는 각 기사의 링크는 실제 사이트 주소가 아니라 `https://news.google.com/rss/articles/CBMi...` 형태의 구글 뉴스 전용 보안/리다이렉션 링크였습니다.

해당 링크를 다이렉트로 구글 번역 프록시(`*.translate.goog`)에 넣고 돌렸을 때, 구글 측의 리다이렉션 정책 및 지오-블로킹으로 인해 **"This translation service isn't available in your region"** 에러 페이지가 노출되며 영어 뉴스를 번역하여 읽을 수 없는 치명적인 UX 실패를 겪었습니다.

## 2. 고려한 대안들 (Alternatives Considered)

### 대안 A: Python 수집기에서 standard HTTP Redirect(Location) 헤더 추적
* *개념:* `urllib` 또는 `requests` 라이브러리로 모든 기사 주소에 요청을 보내 301/302 리다이렉트를 추적해 최종 도달 URL을 수집.
* *한계:* 구글 뉴스는 HTTP Response Header의 `Location`을 주지 않고, 브라우저/클라이언트의 JS(자바스크립트) 실행 또는 메타 리프레시를 이용해 리다이렉션을 발생시킵니다. 따라서 일반 HTTP 통신만으로는 최종 주소를 찾을 수 없고 계속 구글 도메인 URL에 머무릅니다.

### 대안 B: 브라우저 레벨에서 번역 위젯 및 브라우저 자체 번역 기능 안내
* *개념:* 프런트엔드에 구글 번역 셀렉트 박스 위젯을 심어두어 사이트 전체 번역을 유도하거나, 단순 안내 문구를 띄움.
* *한계:* 구글 뉴스 중간 링크를 호출하는 구조는 번역 위젯을 쓰더라도 결과적으로 리전 에러가 발생해 원천적인 문제 해결이 되지 않았습니다. 또한 사용자가 매번 수동으로 브라우저 번역을 켜야 하는 번거로움이 있었습니다.

### 대안 C: googlenewsdecoder 라이브러리 연동 및 신규 뉴스 증분(Incremental) 복호화 적용 [채택]
* *개념:* 구글 뉴스의 최신 `batchexecute` 프로토콜을 모방해 암호화된 URL을 실제 기사 원본 주소로 해독하는 `googlenewsdecoder` 라이브러리를 백엔드 수집 스크립트에 탑재합니다.
* *장점:*
  - 최종 기사 링크(예: TechCrunch, VentureBeat 등)를 직접 저장하므로 구글 번역기에 직접 전달 시 리전 제한 에러를 완벽하게 방지합니다.
  - 매번 150개의 뉴스를 모두 호출하면 Google의 429(Too Many Requests) 차단에 직면하므로, 기존 `news.json`의 기사들과 대조해 **진짜 새로 유입된 신규 뉴스(보통 5~15개)**만 순차적으로 디코딩하는 캐싱 메커니즘을 추가해 안전하게 구동합니다.
* *단점:* `.github/workflows/update_news.yml` 빌드 환경에서 패키지를 설치해야 하는 의존성이 추가됩니다.

## 3. 결정 (Decision)
대안 C를 채택하였습니다. 사용자가 해외 AI 에이전트 동향을 리전 에러 장벽 없이 자연스럽게 번역해서 한글로 읽을 수 있게 하려면, 수집 시점에 **원본 기사 URL을 명확히 확보하는 것**이 필수라고 판단했습니다. 

속도 제한과 차단 이슈는 기존 DB(`news.json`)의 타이틀 기반 증분 캐싱으로 처리해 일일 트래픽 부하를 최소화했습니다.

## 4. 결과 및 영향 (Consequences)
* **장점:**
  - 사용자에게 한글 번역 보기 기능이 매끄럽게 제공됩니다.
  - 뉴스 카드에 원본 언론사 도메인의 원문이 직접 노출되어 사이트의 투명성과 신뢰도가 높아졌습니다.
* **단점 및 트레이드오프:**
  - GitHub Actions 환경의 의존성 빌드 타임이 2~3초 가량 증가했습니다.
  - 깃허브 보안상 워크플로우 파일 수정을 푸시하려면 API 토큰이 아닌 사용자 직접의 `git push`가 요구됩니다.
