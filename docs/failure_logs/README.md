# 🔍 Failure Logs (실패 및 장애 기록)

이 디렉토리는 프로젝트 진행 과정에서 겪은 기술적 실패, 설계 오류, 버그, 트러블슈팅 사례를 솔직하게 기록하는 공간입니다. 
지식은 성공뿐만 아니라 실패의 디버깅 과정에서도 깊어집니다. 이를 문서화하여 향후 동일한 실수를 방지하고 개발자들의 학습을 돕습니다.

## 📝 실패 기록 작성 규칙
* 파일명 형식: `fail-[번호]-[주제-영문-소문자].md` (예: `fail-001-google-translate-region-error.md`)
* 포함해야 할 필수 섹션:
  1. **사건/현상 (Incident/Symptom):** 어떤 문제가 어떤 상태에서 발생했는가?
  2. **원인 분석 (Root Cause Analysis):** 왜 이 문제가 발생했는가? 기술적 원인 규명
  3. **해결 및 시도한 방법 (Mitigation & Troubleshooting):** 어떻게 극복했는가?
  4. **교훈 및 방지 대책 (Lessons Learned & Action Items):** 다음 협업이나 에이전트 작업 시 같은 실수를 방지하기 위해 프로세스/코드를 어떻게 고칠 것인가?
